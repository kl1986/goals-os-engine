# Distribution via runtime plugin + Brain Template, not an installer CLI

The Engine ships as a runtime plugin (Claude Code plugin from a marketplace repo as the reference packaging; other runtimes get equivalent packaging via their Adapters). The Library is a second marketplace of optional plugins. The only thing a user clones is a Brain Template. Upgrade = plugin update, structurally incapable of touching the Brain. Decided 11/07/2026.

**Rejected:** a bespoke installer CLI (`brain init/upgrade`) — better migration control but a second product to maintain; git-native clone-and-pull of all three repos — zero tooling but pushes version pinning and schema migrations onto users, and forked engines can never merge upstream again (the failure mode of most distributed vault templates).

**Amended 13/07/2026** (Wayfinder ticket 04, Cutover alignment map): Library plugins may bundle platform-native artifacts that can't be installed via the standard plugin-update mechanism — e.g. an iOS `.shortcut` file (a "capture source" plugin category), distributable only as a committed export + manual import instructions, never auto-updated. This is a documented exception to "upgrade = plugin update, structurally incapable of touching the Brain": the artifact itself sits outside that update path, though the plugin's real code (receiving/sweep scripts, Adapter config like launchd plists) still updates normally.

**Correction 13/07/2026** (same ticket, on-device test): a hand-authored, unsigned `.shortcut` plist is not importable at all on current iOS — there is no "Allow Untrusted Shortcuts" bypass available for a file arriving via the Files app outside the Shortcuts app itself (confirmed on Kelvin's device). "Committed export + manual import instructions" only works if the export is produced by Apple's own Shortcuts app (Share → Export File, or an iCloud share link) from an already-built, working Shortcut — never fabricated externally by an Adapter or agent. This narrows the exception above: a Library "capture source" plugin bundling a Shortcut requires a human to build it once in the Shortcuts app and export it, not just a spec for an agent to implement from.
